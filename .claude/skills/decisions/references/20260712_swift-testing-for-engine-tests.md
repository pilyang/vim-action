# 엔진 단위 테스트 프레임워크로 Swift Testing 채택

- **결정일**: 2026-07-12

## 결정

모드 엔진(및 이후 추가되는 순수 Swift SPM 타깃)의 단위 테스트 프레임워크로 **Swift Testing**(`@Test`/`#expect`)을 사용한다. Xcode 템플릿이 생성한 UI 테스트 타깃은 XCTest 그대로 둔다.

## 배경·근거 (왜)

엔진 테스트 전략은 "키 시퀀스 → 기대 VimAction" 픽스처 테이블을 대량으로 돌리는 픽스처 기반 테스트다. Swift Testing은 파라미터라이즈드 테스트(`@Test(arguments:)`)가 내장이라 픽스처 하나하나가 독립 테스트 케이스로 리포트되고(실패 지점 즉시 식별), 수동 루프 없이 테이블을 그대로 물릴 수 있다. 첫 테스트를 깔기 전에 정해야 이후 전환 비용이 없어, 엔진 첫 마일스톤 시작 시점에 확정했다.

## 검토한 대안

- **XCTest**: UI 테스트 타깃과 프레임워크가 통일되는 장점이 있으나, 픽스처 테이블을 수동 루프로 돌려야 해 실패 리포팅 단위가 뭉개진다. 파라미터라이즈드 픽스처 테스트가 핵심 워크로드인 이 프로젝트에는 부적합해 기각.
